## 1. 프로젝트 개요

### 1.1 서비스명
**"운세의 모든 것" (가칭)**

### 1.2 서비스 정의
다양한 운세를 한 곳에서 확인할 수 있는 모바일 우선 웹 서비스

### 1.3 목표
- **비전**: 누구나 쉽고 재미있게 운세를 확인할 수 있는 플랫폼
- **미션**: 일상에 작은 위로와 재미를 제공하는 서비스

### 1.4 타겟 사용자
- **주 타겟**: 20-40대 여성
- **부 타겟**: 운세에 관심 있는 모든 연령층
- **페르소나**:
  - 이름: 김지은 (28세, 직장인)
  - 니즈: "출근길에 간단하게 오늘의 운세 보고 싶어요"
  - 행동: 매일 아침 지하철에서 스마트폰으로 확인

## 2. 배경 및 목적

### 2.1 왜 이 서비스를 만드나?

**문제점**:
- 기존 운세 사이트는 UI가 복잡하고 광고가 많음
- 모바일 최적화가 안 되어 사용 불편
- 여러 운세를 보려면 여러 사이트를 방문해야 함

**해결 방안**:
- 깔끔한 모바일 UI
- 한 곳에서 모든 운세 확인
- 빠른 로딩, 광고 최소화

### 2.2 기대 효과

**비즈니스 목표**:
- 런칭 1개월 내 DAU 1,000명
- 3개월 내 MAU 10,000명
- 재방문율 30% 이상

**사용자 가치**:
- 하루를 시작하는 작은 즐거움
- 의사결정에 참고할 수 있는 정보
- 친구들과 공유하는 재미

## 3. 핵심 기능

### 3.1 MVP (필수 기능)

#### 📅 오늘의 운세
**목적**: 매일 방문하게 만드는 핵심 기능

**기능 상세**:
- 별자리 운세 (12개)
- 띠별 운세 (12개)
- 생년월일 입력 시 개인 맞춤 운세
- 종합운, 애정운, 금전운, 건강운, 직장운

**사용자 플로우**:
```
1. 홈 화면 진입
2. 생년월일 입력 (선택) 또는 별자리/띠 선택
3. 운세 결과 확인
4. 공유하기 (선택)
```

#### 🔮 타로 운세
**목적**: 구체적인 질문에 대한 답

**기능 상세**:
- 카드 3장 뽑기
- 질문 입력 (선택)
- 카드별 해석 제공
- 종합 해석

**UI 요소**:
- 카드 뒤집기 애니메이션
- 신비로운 분위기의 배경
- 결과 저장 기능

#### 🎲 사주 운세
**목적**: 깊이 있는 운세 제공

**기능 상세**:
- 생년월일시 입력
- 사주팔자 계산
- 올해 운세
- 월별 운세

#### 💑 궁합 보기
**목적**: 공유와 바이럴을 유도

**기능 상세**:
- 두 사람의 생년월일 입력
- 궁합 점수 (100점 만점)
- 장점/주의점 제공
- 결과 이미지로 저장/공유

## 4. 화면 설계 (모바일 UI)

### 4.1 화면 구조

```
1. 홈 화면
   ├─ 오늘의 운세 (메인)
   ├─ 타로 카드
   ├─ 사주 운세
   └─ 궁합 보기

2. 오늘의 운세 화면
   ├─ 생년월일 입력
   ├─ 별자리/띠 선택
   └─ 운세 결과

3. 타로 카드 화면
   ├─ 질문 입력
   ├─ 카드 선택
   └─ 결과 해석

4. 사주 운세 화면
   ├─ 생년월일시 입력
   └─ 사주 분석 결과

5. 궁합 화면
   ├─ 두 사람 정보 입력
   └─ 궁합 결과
```

## 5. 사용자 스토리

### US-01: 오늘의 운세 확인
```
As a 직장인
I want 출근길에 빠르게 오늘의 운세를 확인하고
So that 하루를 긍정적으로 시작할 수 있다
```

**완료 조건**:
- [ ] 생년월일 입력 시 자동 저장
- [ ] 다음 방문 시 바로 운세 확인
- [ ] 로딩 시간 2초 이내

### US-02: 타로 카드 뽑기
```
As a 운세 애호가

So that 방향성을 얻고 싶다
```

**완료 조건**:
- [ ] 3장 카드 뽑기
- [ ] 카드별 의미 제공
- [ ] 종합 해석 제공

```
As a 20대 여성
I want 친구와 궁합을 보고 공유하고
So that 재미있는 시간을 보내고 싶다
```

**완료 조건**:
- [ ] 두 사람 정보 입력

## 7. 데이터 설계

### 7.1 주요 데이터

#### 사용자 정보 (선택적 저장)
```json
{
  "uuid": "unique-id",
  "birth_date": "1995-03-15",
  "birth_time": "14:30",
  "zodiac_sign": "pisces",
  "chinese_zodiac": "pig"
}
```

#### 운세 데이터
```json
{
  "date": "2026-02-07",
  "zodiac_sign": "aries",
  "fortune": {
    "overall": 4.0,
    "love": 4.5,
    "money": 3.0,
    "health": 4.0,
    "work": 3.5
  },
  "message": {
    "love": "오늘은 좋은 만남이...",
    "money": "작은 행운이...",
    "health": "컨디션 최고!",
    "work": "차근차근..."
  }
}
```

### 7.2 데이터베이스 스키마

```sql
-- 사용자 (선택적)
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    uuid TEXT UNIQUE,
    birth_date DATE,
    birth_time TIME,
    created_at TIMESTAMP
);

-- 운세 기록
CREATE TABLE fortune_history (
    id INTEGER PRIMARY KEY,
    user_uuid TEXT,
    fortune_type TEXT,
    result JSON,
    created_at TIMESTAMP
);

-- 궁합 기록
CREATE TABLE compatibility_history (
    id INTEGER PRIMARY KEY,
    person1_birth DATE,
    person2_birth DATE,
    score INTEGER,
    created_at TIMESTAMP
);
```

## 8. 비기능 요구사항

### 8.1 성능
- **페이지 로딩**: 2초 이내
- **API 응답**: 500ms 이내
- **이미지 최적화**: WebP 포맷, Lazy loading

### 8.2 보안
- **개인정보**: 최소 수집, 암호화 저장
- **HTTPS**: 필수
- **SQL Injection 방지**: ORM 사용

### 8.3 접근성
- **모바일 우선**: 360px 이상 모든 화면
- **터치 친화적**: 버튼 최소 44x44px
- **색상 대비**: WCAG AA 수준

### 8.4 브라우저 지원
- Chrome (최신 2버전)
- Safari (iOS 13+)
- Samsung Internet
