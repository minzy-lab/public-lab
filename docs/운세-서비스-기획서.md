# 📋 종합 운세 서비스 기획서 (PRD)

> PM 역할로 작성된 기획서입니다.
> **버전 2.0 - 종합 운세 우선 전략**

---

## 📌 핵심 전략 요약

### 🎯 단계별 런칭 전략
```
Phase 1 (3주) : 웹 기반 종합 운세 서비스 먼저 런칭
              └─ 사주 운세 (메인), 타로, 궁합, 년간 운세

Phase 2 (2-3주): 카카오톡 연동 데일리 서비스로 리텐션 강화
              └─ 카톡 → 매일 운세 요약 발송
              └─ 웹 → 상세 내용 확인 (카톡 버튼 클릭 시)
```

### 💡 왜 이 전략인가?

**Phase 1의 장점**:
- ✅ **깊이 있는 콘텐츠**: 한 번 보면 오래 참고하는 가치
- ✅ **데이터 확보**: 생년월일시 수집으로 Phase 2 개인화 기반 마련
- ✅ **차별화**: 무료 상세 사주 분석으로 경쟁 우위
- ✅ **신뢰도**: 정확한 분석으로 브랜드 신뢰 구축

**Phase 2: 카카오톡 연동으로 리텐션 극대화**:
- 🔔 **카카오톡 알림톡/친구톡**: 매일 아침 운세 요약 문장 발송
- 🔗 **웹 연결**: "상세 보기" 버튼 클릭 → 웹페이지로 이동
- 📊 **데이터 활용**: Phase 1 사주 데이터 기반 개인화된 일일 운세
- 💡 **사용자 편의**: 카톡만 확인해도 운세 파악, 궁금하면 웹 방문
- 🎯 **리텐션 강화**: 매일 카톡으로 자연스러운 접점 형성

## 1. 프로젝트 개요

### 1.1 서비스명
**"운세의 모든 것" (가칭)**

### 1.2 서비스 정의
다양한 운세를 한 곳에서 확인할 수 있는 모바일 우선 웹 서비스

### 1.3 목표
- **비전**: 누구나 쉽고 재미있게 운세를 확인할 수 있는 플랫폼
- **미션**: 일상에 작은 위로와 재미를 제공하는 서비스

### 1.4 타겟 사용자
- **주 타겟**: 20-40대 여성
- **부 타겟**: 운세에 관심 있는 모든 연령층
- **페르소나**:
  - 이름: 김지은 (28세, 직장인)
  - 니즈: "출근길에 간단하게 오늘의 운세 보고 싶어요"
  - 행동: 매일 아침 지하철에서 스마트폰으로 확인

## 2. 배경 및 목적

### 2.1 왜 이 서비스를 만드나?

**문제점**:
- 기존 운세 사이트는 UI가 복잡하고 광고가 많음
- 모바일 최적화가 안 되어 사용 불편
- 여러 운세를 보려면 여러 사이트를 방문해야 함

**해결 방안**:
- 깔끔한 모바일 UI
- 한 곳에서 모든 운세 확인
- 빠른 로딩, 광고 최소화

### 2.2 기대 효과

**비즈니스 목표**:
- 런칭 1개월 내 DAU 1,000명
- 3개월 내 MAU 10,000명
- 재방문율 30% 이상

**사용자 가치**:
- 하루를 시작하는 작은 즐거움
- 의사결정에 참고할 수 있는 정보
- 친구들과 공유하는 재미

## 3. 핵심 기능

### 🎯 개발 전략
**Phase 1 (MVP)**: 종합 운세 서비스 먼저 런칭
**Phase 2**: 데일리 서비스로 발전

---

### 3.1 Phase 1: MVP - 종합 운세 (필수 기능)

#### 🎲 사주 운세
**목적**: 깊이 있는 종합 운세 제공 (핵심 기능)

**기능 상세**:
- 생년월일시 입력
- 사주팔자 계산
- 올해 운세 (종합운, 애정운, 금전운, 건강운, 직장운)
- 월별 운세 (12개월)
- 년간 흐름 분석

**사용자 플로우**:
```
1. 홈 화면 진입
2. 생년월일시 입력
3. 사주 분석 결과 확인
4. 올해/월별 운세 확인
5. 공유하기 (선택)
```

**왜 사주를 MVP로?**
- 한 번 보면 오래 참고할 수 있는 콘텐츠
- 깊이 있는 정보로 신뢰도 확보
- 생년월일시 수집으로 향후 개인화 기반 마련

#### 🔮 타로 운세
**목적**: 구체적인 질문에 대한 답

**기능 상세**:
- 카드 3장 뽑기
- 질문 입력 (선택)
- 카드별 해석 제공
- 종합 해석

**UI 요소**:
- 카드 뒤집기 애니메이션
- 신비로운 분위기의 배경
- 결과 저장 기능

#### 💑 궁합 보기
**목적**: 공유와 바이럴을 유도

**기능 상세**:
- 두 사람의 생년월일 입력
- 궁합 점수 (100점 만점)
- 사주 기반 궁합 분석
- 장점/주의점 제공
- 결과 이미지로 저장/공유

#### 🌙 년간 운세
**목적**: 장기적인 계획 수립 참고

**기능 상세**:
- 생년월일 기반 올해 운세
- 월별 중요 시기 표시
- 좋은 날, 조심할 날 캘린더
- 주요 이벤트 예측

---

### 3.2 Phase 2: 카카오톡 연동 데일리 서비스 (향후 추가)

#### 💬 카카오톡 일일 운세 (핵심 기능)
**목적**: 매일 접점 형성으로 리텐션 극대화

**플랫폼 전략**:
```
카카오톡 (Push)          웹사이트 (Pull)
      ↓                      ↑
매일 아침 요약 발송  →  상세 내용 확인
(알림톡/친구톡)         (버튼 클릭 시)
```

**기능 상세**:
- **카카오톡 메시지 (매일 아침 7-8시 발송)**
  ```
  🔮 [이름]님의 오늘 운세

  종합운 ⭐⭐⭐⭐ 4/5
  오늘은 좋은 일이 생길 예감!

  💝 애정운: 기대되는 만남
  💰 금전운: 작은 행운 주의

  [상세 보기] ← 웹 링크
  ```
- **개인화**
  - Phase 1에서 수집한 생년월일 기반
  - 사주 데이터와 연동한 정확한 운세
  - 이름 호칭으로 친근감 형성

**웹 페이지 연결**:
- 카톡 버튼 클릭 → 개인화된 웹 페이지 이동
- 상세 운세 내용 (애정/금전/건강/직장)
- 오늘의 행운 숫자/색상
- 타로 카드 추가 뽑기 유도

**사용자 플로우**:
```
1. 사용자: 웹에서 사주 분석 (Phase 1)
2. 사용자: 카카오톡 알림 신청 (선택)
3. 시스템: 매일 아침 카톡으로 운세 요약 발송
4. 사용자: 카톡에서 간단히 확인
5. 사용자: 궁금하면 "상세 보기" 버튼 클릭
6. 시스템: 웹페이지로 리다이렉트 (상세 운세)
```

**기술 구현**:
- **카카오톡 비즈니스 채널** (무료)
  - 알림톡 또는 친구톡 활용
  - 버튼형 메시지 템플릿
  - 개인별 발송 스케줄링
- **웹 랜딩 페이지**
  - URL 파라미터로 사용자 식별
  - 개인화된 상세 운세 표시
  - 추가 기능 이용 유도 (타로, 궁합 등)

**Phase 1과의 연계**:
- Phase 1 사용자에게 카톡 알림 신청 제안
- "매일 아침 운세 받고 싶으세요?" CTA
- 생년월일 데이터 재활용으로 즉시 서비스 가능

---

#### 🎰 럭키 넘버/컬러 (웹 연동)
**목적**: 일상에 재미 요소 추가

**기능 상세**:
- 카톡 메시지에 간단 표시
- 웹에서 상세 설명 및 활용법 제공
- 오늘의 행운의 숫자 (3개)
- 오늘의 행운의 색상 (HEX 코드 포함)
- 추천 로또 번호 (사주 기반)

---

#### 📊 카카오톡 분석 및 최적화
**목적**: 메시지 효과 극대화

**측정 지표**:
- 메시지 오픈율
- 웹 클릭률 (CTR)
- 웹 방문 후 체류 시간
- 추가 기능 이용률 (타로, 궁합)

**최적화 요소**:
- 발송 시간 테스트 (7시 vs 8시 vs 9시)
- 메시지 문구 A/B 테스트
- 이모지 사용 최적화

## 4. 화면 설계 (모바일 UI)

### 4.1 화면 구조 (Phase 1 기준)

```
1. 홈 화면
   ├─ 사주 운세 (메인 - MVP)
   ├─ 타로 카드
   ├─ 궁합 보기
   └─ 년간 운세

2. 사주 운세 화면
   ├─ 생년월일시 입력
   ├─ 사주팔자 분석 결과
   ├─ 올해 운세 (종합/애정/금전/건강/직장)
   └─ 월별 운세 (12개월)

3. 타로 카드 화면
   ├─ 질문 입력
   ├─ 카드 선택
   └─ 결과 해석

4. 궁합 화면
   ├─ 두 사람 정보 입력 (생년월일)
   └─ 궁합 결과

5. 년간 운세 화면
   ├─ 생년월일 입력
   ├─ 올해 전체 운세
   └─ 월별 캘린더 (좋은 날/조심할 날)
```

**Phase 2 추가 화면**:
```
6. 오늘의 운세 화면 (Phase 2)
   ├─ 별자리/띠 선택
   └─ 오늘의 운세 결과
```

## 5. 사용자 스토리

### Phase 1 사용자 스토리

### US-01: 사주 운세 확인 (핵심)
```
As a 운세에 관심 있는 사용자
I want 내 생년월일시를 입력하여 종합적인 사주 분석을 받고
So that 올해와 앞으로의 운세를 깊이 있게 이해할 수 있다
```

**완료 조건**:
- [ ] 생년월일시 입력 및 자동 저장
- [ ] 사주팔자 계산 및 표시
- [ ] 올해 운세 (종합/애정/금전/건강/직장) 제공
- [ ] 월별 운세 (12개월) 제공
- [ ] 결과 저장 및 공유 기능
- [ ] 로딩 시간 3초 이내

### US-02: 타로 카드 뽑기
```
As a 운세 애호가
I want 궁금한 일에 대해 타로 카드를 뽑고
So that 구체적인 질문에 대한 방향성을 얻고 싶다
```

**완료 조건**:
- [ ] 3장 카드 뽑기
- [ ] 카드별 의미 제공
- [ ] 종합 해석 제공
- [ ] 결과 저장 기능

### US-03: 친구와 궁합 보기
```
As a 20대 여성
I want 친구/연인과 궁합을 보고 공유하고
So that 재미있는 시간을 보내고 관계를 이해하고 싶다
```

**완료 조건**:
- [ ] 두 사람 정보 입력 (생년월일)
- [ ] 사주 기반 궁합 점수 계산
- [ ] 궁합 해석 (장점/주의점)
- [ ] SNS 공유 가능 (이미지)

### US-04: 년간 운세 확인
```
As a 계획적인 사용자
I want 올해 전체 운세와 월별 흐름을 보고
So that 중요한 결정을 할 시기를 파악하고 싶다
```

**완료 조건**:
- [ ] 생년월일 기반 년간 운세
- [ ] 월별 캘린더 (좋은 날/조심할 날)
- [ ] 주요 이벤트 예측

---

### Phase 2 사용자 스토리

### US-05: 카카오톡으로 오늘의 운세 받기 (Phase 2)
```
As a 직장인
I want 매일 아침 카톡으로 오늘의 운세 요약을 받고
So that 출근길에 간편하게 확인하고, 궁금하면 웹에서 상세 내용을 볼 수 있다
```

**완료 조건**:
- [ ] Phase 1에서 저장된 생년월일 활용
- [ ] 카카오톡 알림 신청 기능
- [ ] 매일 아침 카톡 메시지 자동 발송
- [ ] 카톡 메시지에 운세 요약 (1-2줄)
- [ ] "상세 보기" 버튼으로 웹 페이지 연결
- [ ] 웹에서 상세 운세 확인 (애정/금전/건강/직장)
- [ ] 사주 데이터 기반 개인화 운세
- [ ] 카톡→웹 이동 시 로딩 2초 이내

### US-06: 카톡 알림 관리 (Phase 2)
```
As a 사용자
I want 카톡 알림을 켜고 끄거나, 발송 시간을 조정하고
So that 내가 원하는 시간에만 운세를 받을 수 있다
```

**완료 조건**:
- [ ] 알림 ON/OFF 설정
- [ ] 발송 시간 변경 (7시/8시/9시 선택)
- [ ] 웹에서 설정 관리 페이지

## 6. 기술 스택

### 6.1 프론트엔드
- **HTML5**: 시맨틱 마크업
